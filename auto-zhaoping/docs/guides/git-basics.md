# Git 基础入门

这份文档放在 `docs/guides/git-basics.md`，用于解释 Git 里最常见的几个概念：分支、提交、回滚、远端和 PR。

## 1. 先记住四个词

### 分支

- 分支就是一条独立的开发线。
- 你在分支上改文件，不会直接影响主线。
- 适合用来做一次任务、一组修复，或者一项新功能。

### 提交

- 提交就是把当前改动保存成一个版本点。
- 每次 commit 都会有一个唯一编号。
- 好的提交应该只做一件事，方便以后回看和回滚。

### 远端

- 远端通常指 GitHub 上的仓库。
- 本地改动要通过 `git push` 推到远端。
- 远端上的分支、PR 和主分支会作为团队协作的共享状态。

### 回滚

- 回滚就是把错误改动撤回。
- 可以撤销未提交改动，也可以回退已经提交的版本。
- 在不确定时，先确认是“撤销本地未提交改动”还是“回退历史提交”。

## 2. 最常见的日常命令

### 看状态

```bash
git status
```

### 看差异

```bash
git diff
```

### 保存改动

```bash
git add .
git commit -m "message"
```

### 推到远端

```bash
git push
```

### 查看历史

```bash
git log --oneline --graph --decorate -10
```

## 3. 在这个仓库里怎么理解

- `main` 通常代表正式主线。
- 我们会在独立分支里完成文档整理、功能修复或测试调整。
- 完成后再通过 PR 合并回主线。

## 4. 什么时候需要小心

- `git add .` 会把当前目录下所有改动都加入暂存区。
- `git commit` 之前最好先看 `git diff`。
- `git push` 之前要确认分支名和远端地址是对的。
- `git reset --hard` 这类命令会丢失改动，除非你非常确定，不然不要用。

## 5. 最短记忆法

- 分支 = 独立工作线
- 提交 = 保存一个版本点
- 远端 = GitHub 上的共享仓库
- 回滚 = 撤销错误改动

如果你只想先会用 3 个命令，先记：`git status`、`git diff`、`git commit`。
