# 腾讯文档写回阶段三设计文档

## 一、项目概述

阶段三负责把阶段二生成的本地结果写回腾讯文档目标表。

本阶段只消费 `results/local-recruiting/latest/`，不再读取腾讯文档提取层，不重新计算 diff，不生成新的本地结果集。它的职责是把阶段二已经稳定产出的 `main-current.csv` 写入腾讯文档，并在写入前保留一个同文档内的备份页，降低覆盖失败的破坏面。

## 二、项目目标

- 只从 `results/local-recruiting/latest/` 读取阶段二结果
- 将 `main-current.csv` 写回腾讯文档目标表
- 写回前先把目标表当前内容复制到同文档内的备份页
- 写回后做读回校验，确认目标表与本地结果一致
- 生成本地运行报告，便于排查写回失败

## 三、已确认范围

- 输入只认 `results/local-recruiting/latest/`
- 目标文档只支持一个明确的 `targetUrl`
- 第一版只做腾讯文档写回，不做 dry-run
- 第一版不做多目标表、不做批量写回
- 第一版保留同文档内备份页，但不做自动回滚

## 四、范围外

- 不再访问腾讯文档源表
- 不重新执行阶段一提取
- 不重新生成阶段二 local results
- 不做自动回滚
- 不做多目标同步
- 不做 diff 级局部更新

## 五、推荐方案

本阶段采用“**独立写回包 + latest 输入 + 备份页保护 + 覆盖写回 + 读回校验**”方案。

### 5.1 方案特征

- 新增独立 workspace 包，例如 `@zhaoping/tencent-docs-writeback`
- 只依赖阶段二输出契约，不依赖阶段一提取内部实现
- 通过腾讯文档浏览器自动化完成备份、覆盖和校验
- 写回成功与失败都保留运行报告

### 5.2 选择原因

- 低耦合，职责清晰
- 只读 `latest/`，输入语义稳定
- 备份页降低覆盖失败造成的损失
- 读回校验能尽早发现目标表写入异常

## 六、总体架构

阶段三由四个职责单元组成：

1. `输入加载层`
2. `腾讯文档会话层`
3. `写回与校验层`
4. `运行报告层`

依赖方向固定为：

- 阶段二 `latest/` -> 输入加载层
- 输入加载层 -> 腾讯文档会话层
- 腾讯文档会话层 -> 写回与校验层
- 写回与校验层 -> 运行报告层

阶段三不应依赖 extractor 的提取逻辑，也不应反向读取阶段二 archive 目录。

## 七、目录结构与产物

### 7.1 输入目录

阶段三唯一输入目录固定为：

- `results/local-recruiting/latest/`

该目录必须至少包含：

- `main-current.csv`
- `summary.json`

并且 `summary.json.status` 必须等于 `success`。

### 7.2 输出目录

阶段三的本地输出根目录建议固定为：

- `results/tencent-docs-writeback/`

其下至少包含：

- `runs/<run-id>/report.json`
- `runs/<run-id>/before-target.csv`
- `runs/<run-id>/after-target.csv`

### 7.3 目标文档备份页

目标文档内保留一个固定备份页，例如：

- `__local_recruiting_backup__`

该备份页由本流程专用，不承载业务写入。

本流程必须在本地保存一个状态文件，例如：

- `results/tencent-docs-writeback/state.json`

状态文件至少记录：

- `backup_sheet_id`
- `backup_sheet_title`

规则固定如下：

- 第一次运行时，如果没有任何本地状态且文档中也没有同名备份页，则创建备份页并记录其内部 sheet id
- 后续运行优先按 `backup_sheet_id` 找回备份页
- 如果本地状态记录存在，但文档中出现同名却不同 id 的 sheet，则视为名称冲突并直接失败
- 如果本地状态记录存在，但对应 id 已不存在且同名页也不存在，则重新创建备份页并更新本地记录
- 如果本地状态缺失，但文档中已经存在同名备份页，则视为外部占用并直接失败，不会覆写它
- 备份页保存的是正式页写回前的整页 tab 克隆，而不是只读可见数据

每次成功写回前，先把正式目标页当前内容复制到该备份页，再刷新正式页。

## 八、输入加载与校验

### 8.1 latest 目录识别

阶段三默认从仓库根目录解析 `results/local-recruiting/latest/`。

仓库根目录同样按 `.git` 向上查找确定，不依赖当前工作目录的偶然位置。

### 8.2 输入文件校验

加载 `latest/` 时，必须校验：

- `main-current.csv` 存在
- `summary.json` 存在
- `summary.json` 可解析
- `summary.status === success`
- `summary.snapshot_date` 存在且非空

若校验失败，本次运行直接终止，不触碰腾讯文档。

### 8.3 CSV 校验

`main-current.csv` 必须是一个合法 CSV，且列顺序完全沿用阶段二输出。

阶段三不重新推导业务字段，只把 `main-current.csv` 当成权威最终结果。

## 九、写回流程

### 9.1 预检

1. 解析仓库根目录
2. 读取并校验 `latest/`
3. 读取目标文档配置
4. 校验腾讯文档登录态和目标表可写权限
5. 加载 `main-current.csv`

### 9.2 备份页 

1. 打开正式目标页
2. 通过文档自身的“复制/克隆 sheet tab”能力，把正式页整页复制到同文档内的备份页 `__local_recruiting_backup__`
3. 只允许对本流程已拥有的 `backup_sheet_id` 做覆盖；任何同名但不同 id 的页都必须视为冲突并失败

备份页只用于保留上一次正式页内容，供人工恢复和排查使用。

### 9.3 覆盖写回 

1. 清空正式目标页可写范围
2. 将 `main-current.csv` 的行写入 A1
3. 读取写回后的正式页内容
4. 与 `main-current.csv` 做逐行比对

正式目标页的可写范围定义为当前表格的整个数据区域，包含标题行和所有已有数据行。写回时必须清除旧值，避免新 CSV 行数或列数更少时留下陈旧单元格内容。读取回验收时以标准化后的表格值为准，不以格式样式为准。

这里的“标准化后的表格值”固定定义为：

- 使用读回接口返回的单元格显示文本作为输入
- 将每个单元格值统一转成字符串
- `null` / `undefined` 视为 `''`
- 仅去掉行尾和列尾的空字符串，不去掉行内空格
- 不对日期、数字、公式做语义转换，按显示文本逐字比较
- 以标准化后的二维字符串数组逐行逐列比较

### 9.4 完成

1. 写本地运行报告
2. 退出成功

## 十、失败处理

### 10.1 总体原则

失败必须是显式的、可定位的、不可静默吞掉的。

### 10.2 硬失败场景

以下场景直接终止：

- `latest/` 不存在或不完整
- `summary.status !== success`
- CSV 解析失败
- 腾讯文档未登录或无写权限
- 目标文档不可读/不可写
- 备份页写入失败
- 正式页写回后读回校验失败
- 运行时页面脚本或浏览器异常

### 10.3 失败语义

阶段三不承诺自动回滚。

原因是腾讯文档没有可靠的事务接口；一旦正式页开始清空或写入，失败时只能依靠备份页进行人工恢复。

因此失败后的行为固定为：

- 保留备份页
- 保留本地运行报告
- 不再继续后续写入步骤
- 退出非零码

## 十一、运行报告设计

每次运行都应写出 `report.json`，至少包含：

- `status`
- `run_id`
- `input_dir`
- `snapshot_date`
- `target_url`
- `backup_sheet_name`
- `main_row_count`
- `failure_stage`（失败时）
- `failure_reason`（失败时）
- `suggested_action`（失败时）

成功时报告应记录：

- 备份页是否成功刷新
- 目标页写回后的行数
- 读回校验结果

失败时报告应记录：

- 失败发生在预检、备份、写回还是校验阶段
- 失败前已经完成了哪些步骤

## 十二、测试设计

### 12.1 单元测试

至少覆盖：

- `latest/` 输入校验
- `summary.json` 成功态校验
- CSV 读取与行数统计
- 报告对象构建

### 12.2 编排测试

使用浏览器层和腾讯文档层的 stub/mock，验证：

- 成功路径会先备份再覆盖
- 预检失败时不会触碰目标页
- 写回失败时会停在失败阶段并写报告
- 读回不一致会被判定为失败
- 备份页缺失、被删除或被改名时会重新创建并写入整页副本

### 12.3 手工验收

在真实腾讯文档环境中验证：

- 目标页写回后内容与 `latest/main-current.csv` 一致
- 备份页保存的是写回前内容
- 失败时不会继续执行后续步骤

## 十三、验收标准

- 只能从 `results/local-recruiting/latest/` 读取输入
- 只写一个腾讯文档目标表
- 写回前会保留同文档内备份页
- 写回后会做读回校验
- 失败时保留备份页和运行报告
- 全量测试可通过
